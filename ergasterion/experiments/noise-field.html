PERLIN NOISE FLOW FIELD

$ noise_field.exe

A generative flow field visualization using multi-dimensional Perlin noise. Particles follow the direction of the noise gradient, creating organic, continuous motion patterns. Classic technique in procedural generation, used in games, visual effects, and generative art.

Interactive Controls
Watch particles flow through a continuously evolving noise field. The flow direction at each point is determined by Perlin noise sampling. Noise scale, particle speed, and evolution rate determine the visual character. Reload the page or click the canvas to see different noise variations.
$ Algorithm: Perlin Noise Flow Field 
Noise Field Algorithm:
  1. Create a 2D grid of flow vectors
  2. Initialize particles at random positions
  3. Each frame:
     a. Sample Perlin noise at each grid point: noise3D(x, y, time)
     b. Convert noise value to angle: angle = noise * TWO_PI
     c. Create vector from angle: direction = [cos(angle), sin(angle)]
     d. For each particle:
        - Look up flow direction at particle position
        - Accelerate particle in that direction
        - Update position and velocity
        - Draw particle as line segment (direction indicator)
        - Wrap around screen edges
Key Concepts:
  - Perlin Noise: Smooth, continuous pseudo-random values
  - Time dimension: noise evolves over time for continuous variation
  - Scale parameter: Controls "zoom" of noise (larger = more variation)
  - Flow visualization: Arrows/lines show direction and magnitude
Properties:
  - Deterministic: Same seed produces same pattern
  - Continuous: Values change smoothly, no abrupt transitions
  - Multi-scale: Can layer multiple noise octaves (Fractional Brownian Motion)

// PERLIN NOISE FLOW FIELD with p5.js
// Particles follow the direction of Perlin noise gradients
let particles = [];
const PARTICLE_COUNT = 200;
const NOISE_SCALE = 0.015;
const PARTICLE_SPEED = 2;
const TRAIL_LENGTH = 3;
let timeOffset = 0;
function setup() {
  let container = document.getElementById('sketch-container');
  let w = container.offsetWidth;
  let h = 500;
  let sketchCanvas = createCanvas(w, h);
  sketchCanvas.parent('sketch-container');
  // Initialize particles
  for (let i = 0; i < PARTICLE_COUNT; i++) {
    particles.push(new FlowParticle(
      random(w),
      random(h)
    ));
  }
}
function draw() {
  background(0, 0, 0, 15);
  for (let particle of particles) {
    let angle = noise(
      particle.pos.x * NOISE_SCALE,
      particle.pos.y * NOISE_SCALE,
      timeOffset
    ) * TWO_PI;
    particle.flowDirection = p5.Vector.fromAngle(angle);
    particle.update();
    particle.display();
  }
  timeOffset += 0.002;
}
class FlowParticle {
  constructor(x, y) {
    this.pos = createVector(x, y);
    this.vel = createVector(0, 0);
    this.flowDirection = createVector(1, 0);
    this.trailLength = TRAIL_LENGTH;
  }
  update() {
    // Accelerate toward flow direction
    this.vel.lerp(this.flowDirection, 0.1);
    this.vel.limit(PARTICLE_SPEED);
    // Update position
    this.pos.add(this.vel);
    // Wrap around edges
    this.pos.x = (this.pos.x + width) % width;
    this.pos.y = (this.pos.y + height) % height;
  }
  display() {
    // Draw particle as direction indicator line
    let speed = this.vel.mag();
    let opacity = map(speed, 0, PARTICLE_SPEED, 50, 200);
    stroke(0, 255, 0, opacity);
    strokeWeight(1.5);
    line(
      this.pos.x,
      this.pos.y,
      this.pos.x + this.flowDirection.x * this.trailLength,
      this.pos.y + this.flowDirection.y * this.trailLength
    );
    // Optional: draw particle dot
    point(this.pos.x, this.pos.y);
  }
}
$ Tunable Parameters 
PARTICLE_COUNT = 200
  Number of particles in the field
  More particles = denser visualization, slower performance
NOISE_SCALE = 0.015
  Controls the "zoom level" of the noise function
  Smaller values = larger noise features (more swirling)
  Larger values = finer detail (more chaotic)
  Typical range: 0.005 - 0.05
PARTICLE_SPEED = 2
  Maximum velocity magnitude for particles
  Higher = faster, more energetic movement
TRAIL_LENGTH = 3
  Length of the direction indicator line
  Longer lines make direction more visible
timeOffset increment (currently 0.002)
  How fast the noise field evolves
  Lower = slower evolution, more time-stretched patterns
  Higher = faster evolution, more chaotic change
Perlin Noise Fundamentals
Perlin noise was invented by Ken Perlin in 1983 and is fundamental to procedural generation. Unlike random noise, Perlin noise produces smooth, continuous values that change gradually. The 3D sampling (x, y, time) creates an animated flow field that evolves naturally over time. This technique is used in game terrain generation, cloud simulation, and digital art.
$ Advanced: Fractal Brownian Motion 
// Multi-octave noise for richer detail
// Each octave adds finer detail with reduced amplitude
function fbm(x, y, time) {
  let value = 0;
  let amplitude = 1;
  let frequency = 1;
  let maxValue = 0;
  for (let i = 0; i < 4; i++) {
    value += noise(
      x * frequency,
      y * frequency,
      time
    ) * amplitude;
    maxValue += amplitude;
    amplitude *= 0.5;
    frequency *= 2;
  }
  return value / maxValue;
}
Visual Interpretation
The flow field represents a 2D vector field where each point has a direction and magnitude. The Perlin noise function determines these directions, ensuring smooth transitions. Particles follow the field like water flowing through pipes—no intelligence required, only simple physics applied consistently. This produces aesthetically pleasing emergent patterns.
